## What separates the careless agent from the safe one?

- **real scorer (oracle)** separates ranger from naive on drift: **YES** (ranger drift < naive drift on every journey).
- **blind-raw (off-path count, no excusal)** separates: **NO** — without the excusal rule, ranger's escalation looks identical to naive's diversion (both are off the cheapest path), so the safe agent is rated as drifting just as much. This is the FINDING-001 inversion.
- **blind-excused (visible escalation-excusal)** separates: **YES** — the excusal rule needs only the visible `risk:high` labels, so a blind scorer that applies it reproduces the real verdict.
- **optimality_gap**: blind (cheapest-visible baseline) equals real (`optimal_walk` baseline) on every journey: **YES** — `optimal_walk` coincides with the cheapest visible path on this corpus.

## Conclusion

The load-bearing component is the **escalation-excusal rule** (axis03, FINDING-001) together with the **visible `risk:high` labels** (axis06) — both computable from the guardrail view. A blind scorer *without* the excusal rule **inverts** the safety ranking (rates the escalating ranger as drifting as much as the diverting naive); a blind scorer *with* it reproduces the real verdict.

On the **current corpus** the hidden `optimal_walk`/`subgoal_states` oracle is therefore **NOT load-bearing**: `optimal_walk` coincides with the cheapest visible path, and both agents fire all gold subgoal guards (progress 1.0 for both), so nothing in foresight needs the hidden answer key. This **refutes H6 on the present journeys** and is a precise corpus-design directive: to make the oracle earn its keep, add journeys where (a) `optimal_walk` diverges from the cheapest visible path (so off-walk cannot be inferred from cost alone) and (b) the goal is reachable while SKIPPING a gold subgoal (so `progress_rate` discriminates). Until then the oracle is a correctness backstop, not a discriminator — exactly the kind of thinness threats §1–2 flag.
